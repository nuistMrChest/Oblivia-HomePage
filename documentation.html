Type System
Currently, there are five built-in Rvalues: number,string,array,object and reference.

Number

In this version, number is basically float in C, which means in calculation, it may perform weirdly, we will fix this in future versions. Additionally, we used fmod to make it support modulo calculation.

String

We don't have char here, but we have string. In this language, once created, you cannot modify the char inside the stirng. But you can still visit single char, which is acually one charecter string, and you can apply strings tu the end of a string to create sub strings.
There are supported escapes, which are: 
\n \" \a \b \f \r \t \v \\
Remenber, \' is ilegal, bucause ' can already exist in string legally.

Array

We also support array. An array is, ofcourse, a contiguous block of memory. You can contain different types of data in a single array, we will talk about whats going on later. Unlike other script language, you cannot change the length of the array once created. Visiting the array element is just like other languages, using [ ], but we have an operator to do that as well.

Object

Objects are important to describe complicated data structure.
Since this is a dynamic language, class or struct is not needed. For now, you can initialize an enmpy object and add attribute later.

Reference

Reference here is acutally kinda like pointers, but easier to use, and safer.
It must be initialized with a Rvalue pointed. If the Rvalue dies, the reference will be set at a "nullptr" status.
Then, there are three Lvalues: variable,array element, object attribute. To storage a value, you have to combine it with a Lvalue, or it will just be a literal, just like other languages.

Variable

The variables in this language is dynamic, which means is can be given different types of value in a same scope. A variable must have a name and a value. If not assigned when initialized, it will be a number 0 by default. Up until this document is written, you can use key words as variable name, but it is not recommended, we will remove this feature later. A legal identifier only contains'a'-'z' , 'A'-'Z' and '_'. Number is not allowed in identifier.

ArrayElement

Previously, we've talked about array, which is a contiguous block of memory. But to be more precise, it does not contain data directly. It only contains array element, which is why it can store different types. The operator used to visit element of array is actually giving you the array element. Just like variable, it is number 0 by default.

Object Attribute

Just like array, an object is a contiguous block of memory, containing attributes and indexed by identifier.
Expression
To introduce expression, we have to introduce operators.
Supported operator: + - * / % = == != > < >= <= && || ! # -> += -= *= /= %=
Note: # is the operator to visit array element, which we've mentioned previously. When you write a[0], the tokenizer ancually turns it into a#(0), then the expression calculation system gives you the first array element contained in the array conmined with a. -> is the operator to visit object attribute, just like # .More detailed information will be given when object is fully implemented. / is divition of float. Because number is float in c, 3/2 actually gives you 1.5 instead of 1. % is fmod in math.h in libc, so 3%2 is still 1.
Number can be used as bool, 0 is false, any number that is not 0 is true. The trues given by bool operators (&& ! || == != > < >= <=) are number 1. And still, since number is float, sometimes bool calculation may not function properly.
Of coures, these operators have priorities. 

8: # ->
7: !
6: * / %
5: + -
4: > < >= <=
3: == !=
2: &&
1: ||
0: = += -= *= /= %=
You can also use parens inside an expression.
The operators have got associativity.
The associativity of ! = += -= *= /= %= is right, and everyone else's is left.
A thing to remenber is that all the assignment here are deep copy, all the compair here are deep compair, so you don't have to guess like what we used to do in python or java.
What makes an expression legal? When the checker gets an expression tokens, it generates a same tokens you gave it, and it removes all the unary operators, which is actually just ! . Then, for the checker, there are only to types of token: value and operator. identifier , numebr and string literal is seen as value, while operators are just operator. Now, a ilegal expression must follow two rules: value and operator appears alternately, and it must start and end with value, which means it must look like: 

 valte operator value operator ...... operator value
When encontered paren, the things inside the paren will be checked following the two rules above, then the paren, as well as every thing inside, will be removed and replaced with a value: 

value operator(value operator value)operator value 
=> value operator value operator value
Then, when all the binary operators are checked, the checker goes back to the original tokens and check the unary operators. Here, the operator ! must be followed with a value, paren ( or another !.
So as you can see, a ilegal expression is not necessarily calculatable. For instance, 1=1 is a legal expression, but crashes when calculated.
Asaide form normal numeric and boolean calculation, there are some special usage of the operators.

String

you can use + to build a larger string, or use += to append strings, like: 

"a"+"b";
let a="hello ";
a+="world";

Object

For now, there is no object literals, so all the object are empty when initialized. So, we will use + or += to add attributes. 

let a@;
let b=a+c;
Now a is sitll an empty object, but b is an object with an attribute called c.
Like variables, attributes are assigned with number 0 by default.
If you really want to add attribute for a, try this: 

let a@;
a+=c;
Now a is an object with an attribute called c.
Statement
Now we'v entered the core feature of the grammer.
There are three types of statements:normal statement,expression statement,statement block.
A statement must end with ; or wrapped with {}. {} means it is a block. For those who ends with ; , if it starts with a key word, it's a normal statement, otherwise it's an expression statement.

Block

A block must contain more than one statement, which means {} is not a legal statement. When executed, all the statements contained will execute in order. The block also has an important duty: defining scope. Blocks control the life span of variables, and variables control the life span of all the data stored. It is one of the few ways to manage your ram in this language.
Let's talk more about the scope. It works just like every other language: 

{
    let a=9;
    {
        let a=8;
        print a;
    }
    print a;
}
prints: 

89

Expression

An expression statement is legal only when it contains a legal expression and a ; . When executed, it simply calculate the expression, even the result is stored in nowhere. Although things like 1+1; is permitted, but most time it is used to change value of reference types or call functions in the future.

Let

Let statement is used to create variable. It can be used in the following ways: 

let a;
let b=1+9;
let c="giao";
let d#3;
let e[9];
let f@;
The first exaple created a variable with nothing, which, as we mentioned before, will be given number 0.
The second and third examples show that it can be initialized with the result of an expression.
The forth and fifth examples show how to initialize array.
The last example shows how to initialize an empty object.

Print

What a print statement does is calculate an expression and print the result to stdio.
For example, the good old: 

print "Hello World!\n";

Scan

what a scan statement does is also simple:read whatever you inputed as a string, canculate the expression and assign the string to t resule. The result must be assignable, or the program will crash. 

scan a;

If

To implement complex algorithm, we need if. The way to use if is: 

if expr:stat
expr is an expression, and stat is a statement. The result of expr must be a number, which we use as bool here, or the program crashes. If the result is not 0, stat will be executed.

Else

Else statement is used to cooperate with if. If in a block, there must be an if above in the same block, or the program crashes. It will look for the closest if above and link with it. This is how we use it: 

else stat
If the expr of the linked if is 0, stat will be executed.
In some situation else will appear not in a block, then it is legal even if there is no if above, it is legal, even thougn it does absolute nothing.

While

To be a fully functional programing language, we also need loop, which is implemented as while statement, used as: 

while expr:stat
expr is an expression with number result, stat is a statement.
Like if, when executed, expr will be calculated, if true, stat will be executed, then starts the loop. Each time, expr will be calculated, if true, execute stat, and the loop goes on, if false, loop ends.

Break

Break statament is used simply as: 

break;
It can appear anywhere where a statement can appear. What it does is end current block execution and send a stop message. The message passes down all the way to the first while, the the while loop ends what ever the expr is.

Continue

Continue works just like break: 

contunue;
It ends block execution and send continue message to the while, and while jump to the next turn of loop no matter the stat.

Execute

This is a pretty interesting statement, used as: 

execute expr;
Expr is an expression. The result must be a string, and what the string contains must be a legal statement of Oblevia. meaning that: 

execute "print\"hello\";";
prints hello.
This means that you can store code as data. What you can do is to add code while the program is running, printing the variables, which makes it easy to debug.

Include

This is how we split our code into more than one source file. 

include expr;
The result of expr must be a string and contains a legal path to an oblivia source file.
If a file is included instead of run directly by interpreter, it don't have to be a single legal statement, for example: 

let a=0;
let b[10];
let c@;
is a legal file to be included, but cannot run by interpreter.
The base scope level of the file starts by the scope level of the include statement. Like, if the level of the include statement is 3, than inside the example above, a, b and c are in scope level 3, and so on.
Interpreter
The interpreter runs in two mods: no argument means REPL mode which enteres REPL, with one argument as source code path means running a file.

REPL

In the REPL, you can input REPL commands or Oblivia statement. Currently there are six commands: 

/exit
/help
/list_variables
/scope
/scope_up
/scope_down
If you input a statament, it tokenize, check legal, build satement and execute, just like every other REPL. Note: else, break and continue does absolute nothing if inputted directly to REPL.

Running a file

If you run a file, code inside the source file will be tokenized, checked, built and executed. Everything inside a file must be a legal statement as a whole, meaning that: 

print "Hello World!\n";
is a legal program, and: 

{
    let a=100;
    while a>0:{
        print a;
        print "\n";
        if a%2==0:{
            print "AAAAA\n";
        }
        else{
            print "BBBBB\n";
        }
        if a==50:{
            break;
        }
        a=a-1;
    }
}
is also a legal program.
